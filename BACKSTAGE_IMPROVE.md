## Phase 1: 핵심 워크플로우 강화

### 1.1 프로젝트 생성 마법사 (Project Creation Wizard)
**현재 문제:** StartProjectModal이 기본적인 정보만 입력 가능

**개선 방향:**
- [ ] 단계별 프로젝트 설정 (3-4단계)
  - Step 1: 기본 정보 (제목, 설명, 장르)
  - Step 2: 프로젝트 구조 (템플릿 선택)
  - Step 3: 팀 구성 (역할 정의, 멤버 초대)
  - Step 4: 일정 설정 (마일스톤, 마감일)
- [ ] 장르별 프로젝트 템플릿
  ```
  K-Pop 템플릿:
  ├── 1. 기획 (콘셉트, 레퍼런스)
  ├── 2. 작곡 (데모, 탑라인)
  ├── 3. 편곡 (세션 녹음)
  ├── 4. 보컬 녹음
  ├── 5. 믹싱
  ├── 6. 마스터링
  └── 7. 발매 준비
  ```
- [ ] AI 추천 워크플로우 (장르/규모에 따른 자동 제안)

### 1.2 고급 칸반 보드
**현재 문제:** 기본적인 드래그 앤 드롭만 지원

**개선 방향:**
- [ ] 커스텀 컬럼 추가/삭제/이름 변경
- [ ] 컬럼별 WIP(Work In Progress) 제한
- [ ] 카드 필터링 (담당자, 우선순위, 마감일)
- [ ] 카드 라벨/태그 시스템
- [ ] 서브태스크 지원
- [ ] 자동화 규칙 (예: "마스터링 완료" → 자동으로 "발매 준비"로 이동)

### 1.3 타임라인 & 간트 차트
**현재 문제:** 프로젝트 일정 관리 기능 부족

**개선 방향:**
- [ ] 간트 차트 뷰 (GanttChartView.tsx)
- [ ] 마일스톤 관리
- [ ] 의존성 표시 (A 태스크가 끝나야 B 시작)
- [ ] 크리티컬 패스 하이라이트

## 기술 스택 고려사항

### 프론트엔드
- React + TypeScript (현재)
- Zustand/Redux (상태 관리 강화)
- React Query (서버 상태)
- Framer Motion (애니메이션)
- TanStack Virtual (대용량 리스트)

### 백엔드 (향후)
- Node.js + Express / Nest.js
- PostgreSQL (관계형 데이터)
- Redis (캐싱, 실시간)
- S3 (파일 스토리지)
- WebSocket (실시간 협업)

### AI/ML (향후)
- OpenAI API (텍스트 생성)
- Replicate / Hugging Face (음악 AI)
- Whisper (음성 인식)
